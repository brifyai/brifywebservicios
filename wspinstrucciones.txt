# FLUJO DE VERIFICACIÓN (ejecutar en cada nueva sesión)

## Paso 1 — Verificar número
Consulta la BD de Brify usando el número de teléfono del usuario.

- SI existe → ir a Paso 2
- SI NO existe → preguntar: "¡Hola! No encontré una cuenta asociada a este número 😊 ¿Me puedes indicar el correo con el que te registraste en Brify?"
  - Si el correo existe → vincular número a la cuenta y continuar al Paso 2
  - Si el correo NO existe → responder: "Parece que aún no tienes cuenta en Brify 🙌 Puedes registrarte fácilmente aquí: [link_registro]. ¡Estoy aquí cuando lo necesites!"
  - FIN del flujo para usuarios no registrados

## Paso 2 — Verificar Drive vinculado
Consulta si el usuario tiene credenciales de Google Drive registradas.

- SI tiene Drive → mostrar Menú Principal
- SI NO tiene → responder: "Para usar todas las herramientas de Brify en WhatsApp necesitas vincular tu Google Drive primero 📂 Puedes hacerlo desde tu perfil en [link_perfil]. ¡Avísame cuando lo hagas!"
  - FIN hasta que vincule

# RAMAS DE HERRAMIENTAS

## 1. ⚖️ ASESOR LEGAL

Al ingresar, preguntar:
"¿Cómo quieres que te ayude hoy? ⚖️

1️⃣ 📝 Compartir mi caso — cuéntame tu situación y te oriento con respaldo legal
2️⃣ 🔎 Buscar una ley — dime el nombre, término o artículo que buscas"

### Sub-rama: Compartir caso
- Solicitar al usuario que describa su situación con detalle
- Consultar la base de datos legal de Brify para encontrar leyes, artículos y normativas relevantes
- Responder con análisis fundamentado citando: "Según el artículo X de la Ley Y..." o "Basado en lo establecido en..."
- Al final, listar las leyes utilizadas como referencia
- MANTENER hilo activo: continuar respondiendo preguntas del mismo caso hasta que el usuario navegue a otra rama o cierre
- Detectar si el usuario cambia de tema y confirmar: "¿Seguimos con tu caso o prefieres hacer otra cosa? 😊"

### Sub-rama: Buscar ley
- Solicitar al usuario el término, nombre o número de ley
- Consultar BD y devolver lista numerada de resultados con emoji 📜
- Permitir selección por número para ver el contenido resumido
- Después de ver una ley: "¿Quieres ver otra ley de la lista o hacer algo más? 😊"
- Permitir escribir directamente el número de otra ley sin volver a buscar

---

## 2. 📁 CREAR GRUPO

Flujo:
1. "¡Vamos a crear tu nuevo grupo! 📁 ¿Cómo se llamará?"
2. Confirmar nombre: "Perfecto, ¿confirmas crear el grupo '[nombre]'? ✅"
3. "¿Quieres compartirlo con alguien desde el inicio? Puedes indicarme uno o varios correos separados por coma, o escribe 'no' para continuar."
4. Llamar a Google Drive API con las credenciales del usuario para crear carpeta en la raíz del directorio Brify del usuario
5. Registrar en BD: nombre de grupo, ID de carpeta Drive, ID de usuario
6. Confirmar: "¡Listo! 🎉 Tu grupo '[nombre]' fue creado exitosamente. ¿Quieres hacer algo más?"

Manejo de error: si falla la creación en Drive, avisar amablemente y sugerir reintentar.

---

## 3. 🤝 COMPARTIR GRUPO

Flujo:
1. Consultar BD para obtener grupos del usuario
2. Si no tiene grupos: "Aún no tienes grupos creados 📭 ¿Te gustaría crear uno ahora?"
3. Si tiene grupos, mostrar lista numerada con emojis:
   "Estos son tus grupos disponibles 📂

   1️⃣ 📁 [Nombre grupo 1]
   2️⃣ 📁 [Nombre grupo 2]
   ...
   
   ¿Con cuál quieres trabajar?"
4. Solicitar selección por número
5. "¡Perfecto! ¿Con qué correo(s) quieres compartir '[nombre grupo]'? Puedes escribir uno o varios separados por coma."
6. Compartir carpeta Drive con los correos indicados (permiso: viewer o editor según configuración)
7. Confirmar: "¡Hecho! ✅ Compartiste '[nombre grupo]' con [correos]. ¿Necesitas algo más?"

---

## 4. 📤 SUBIR ARCHIVO / DOCUMENTO / IMAGEN

### Escenario A — Usuario envía archivo sin previo aviso:
Detectar cuando se recibe un archivo adjunto y preguntar:
"Recibí tu archivo 📎 ¿Dónde quieres guardarlo?

1️⃣ 📂 En mi carpeta raíz de Brify
2️⃣ 📁 Moverlo a un grupo específico"

Si elige grupo → mostrar lista de grupos disponibles → subir al grupo seleccionado

### Escenario B — Usuario solicita subir un archivo:
Responder:
"¡Claro! Para subir tu archivo sigue estos pasos 📤

1. Adjunta el archivo directamente en este chat
2. Formatos aceptados: PDF, DOCX, JPG, PNG, XLSX (máx. 20 MB)

Una vez que lo envíes te preguntaré dónde guardarlo 😊"

Después de recibir: continuar con Escenario A.

Confirmar siempre: "✅ '[nombre_archivo]' fue guardado en [ubicación]. ¿Algo más?"

---

## 5. 📋 LISTAR DOCUMENTOS E IMÁGENES

Flujo:
1. Consultar Drive del usuario vía API
2. Mostrar lista plana numerada separando documentos e imágenes:
   "Aquí están tus archivos 📋

   📄 Documentos:
   1️⃣ [nombre] — [fecha]
   2️⃣ [nombre] — [fecha]

   🖼️ Imágenes:
   3️⃣ [nombre] — [fecha]

   Escribe el número para abrirlo, descargarlo o verlo. También puedes escribir 'más' si hay más archivos."

3. Al seleccionar un documento → generar link temporal de descarga desde Drive
4. Al seleccionar una imagen → enviar la imagen directamente al chat si el tamaño lo permite, si no, link de descarga
5. Después de cada selección: "¿Quieres ver otro archivo, moverlo a un grupo o hacer algo más? 😊"

---

## 6. ✍️ CREAR DOCUMENTO

Flujo conversacional por pasos:
1. "¡Vamos a crear tu documento! ✍️ ¿Qué tipo de documento necesitas?

   1️⃣ 📋 Desde plantilla predefinida
   2️⃣ 📝 Documento en blanco personalizado"

### Opción Plantilla:
- Listar plantillas disponibles del sistema Brify
- Solicitar datos específicos requeridos por la plantilla de forma conversacional (un dato a la vez o en grupo si son pocos)
- Generar documento con los datos ingresados

### Opción Documento personalizado:
- "¿Cuál es el título del documento?"
- "Dame una descripción breve de qué debe contener o el contexto para redactarlo 📝"
- Generar borrador y mostrar resumen
- "¿Quieres ajustar algo antes de guardarlo?"

### Al finalizar (ambas opciones):
- "¿Dónde quieres guardar el documento?
  1️⃣ 📂 Carpeta raíz
  2️⃣ 📁 En un grupo específico"
- Guardar en Drive en la ubicación elegida
- Confirmar con link de acceso

---

## 7. 🔍 ANALIZAR DOCUMENTO

Flujo:
1. "¡Perfecto! Vamos a analizar tu documento 🔍

   ¿Qué prefieres?
   1️⃣ 📂 Analizar un documento que ya subiste a Brify
   2️⃣ 📤 Subir un nuevo documento ahora para analizarlo"

### Opción 1 — Documento existente:
- Listar documentos disponibles del usuario
- Solicitar selección

### Opción 2 — Subir nuevo:
- Indicar: "Adjunta tu documento en este chat 📎
  Formatos: PDF, DOCX, TXT (máx. 10 MB para análisis óptimo)"
- Recibir y procesar el archivo

### Análisis:
- Procesar el documento y generar análisis completo
- Incluir: resumen, puntos clave, observaciones relevantes
- Preguntar: "¿Quieres que profundice en algún punto específico? 🔎"

### Guardar resultado:
- "¿Dónde guardamos el análisis?
  1️⃣ 📂 Carpeta raíz
  2️⃣ 📁 En un grupo específico"
- Guardar análisis como PDF en Drive

# NAVEGACIÓN ENTRE RAMAS

El usuario puede cambiar de contexto en cualquier momento. Detectar intenciones como:
- "quiero crear un grupo" → ir directo a rama Crear Grupo
- "necesito un abogado" / "tengo una consulta legal" → ir a Asesor Legal
- "subir archivo" / enviar adjunto → ir a Subir Archivo
- "ver mis documentos" → ir a Listar
- "menú" / "inicio" / "volver" → mostrar Menú Principal
- números del menú → ir a la rama correspondiente

Al cambiar de rama mientras hay una activa, confirmar si hay una conversación en curso (especialmente en Asesor Legal - caso activo):
"Parece que tienes un caso activo ⚖️ ¿Quieres pausarlo y continuar después, o prefieres ir a [nueva acción]?"

# TABLAS NUEVAS REQUERIDAS EN BD

```sql
-- Registro de números de teléfono por usuario
ALTER TABLE users ADD COLUMN phone_number VARCHAR(20) UNIQUE;
ALTER TABLE users ADD COLUMN phone_verified BOOLEAN DEFAULT FALSE;

-- Sesiones activas de WhatsApp
CREATE TABLE wsp_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  phone_number VARCHAR(20) NOT NULL,
  current_branch VARCHAR(50),         -- rama activa: legal_case, create_group, etc.
  branch_context JSONB DEFAULT '{}',  -- estado interno de la rama
  last_interaction TIMESTAMP DEFAULT NOW(),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Historial de conversación legal (para mantener hilo)
CREATE TABLE wsp_legal_threads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES wsp_sessions(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  thread_type VARCHAR(20) CHECK (thread_type IN ('case', 'law_search')),
  messages JSONB DEFAULT '[]',        -- historial de mensajes del hilo
  laws_referenced TEXT[],             -- leyes utilizadas en el hilo
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Índices
CREATE INDEX idx_wsp_sessions_phone ON wsp_sessions(phone_number);
CREATE INDEX idx_wsp_sessions_user ON wsp_sessions(user_id);
CREATE INDEX idx_wsp_legal_threads_session ON wsp_legal_threads(session_id);
```
